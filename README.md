# standalone-witness

A minimal standalone witness HTTP daemon for the Attesta transparency
log network. Loads a single secp256k1 PEM key + the network bootstrap
document, serves `POST /v1/cosign` on the configured port.

Designed for multi-instance deployments where a writer ledger needs
N external witnesses to drive a real K-of-N quorum without spinning
up N full ledger processes.

## What it is not

This is NOT a full ledger. It does NOT participate in gossip, hold
a Postgres connection, write tiles, run a builder loop, or accept
admissions. It exclusively serves cosignature requests.

For a full witness deployment with rotation + gossip + persistence,
run a regular ledger process configured with the witness role.

## Architecture

The cosign handler wraps the Attesta SDK's
`cosign.NewWitnessHandler` with a local monotonicity guard that
refuses to cosign a `tree_size` smaller than this process's
`lastSignedSize`. So any test that runs against this binary
exercises the same code path as a production K=N witness fleet.

Network binding: the witness's `NetworkID` is derived from the
network bootstrap document (`network.BootstrapDocument.IDs()`).
Requests carrying a different `network_id` are rejected with 403 by
the SDK handler.

## Build

```sh
go build -o ./bin/standalone-witness .
```

Or via `go install`:

```sh
go install github.com/clearcompass-ai/standalone-witness@latest
```

## Run

```sh
./bin/standalone-witness \
    -addr :8081 \
    -key-file .run/witnesses/witness-1.pem \
    -bootstrap .run/network-bootstrap.json
```

Flags:

| Flag         | Required | Description                                                            |
| ------------ | -------- | ---------------------------------------------------------------------- |
| `-addr`      | no       | HTTP listen address. Default `:8081`.                                  |
| `-key-file`  | yes      | Path to the witness EC private key in PEM form.                        |
| `-bootstrap` | yes      | Path to the network `BootstrapDocument` JSON (drives the `NetworkID`). |

The keypair + bootstrap document are generated by the Attesta
Ledger's `cmd/init-network` tool.

## Endpoints

| Method | Path         | Description                                                                                       |
| ------ | ------------ | ------------------------------------------------------------------------------------------------- |
| POST   | `/v1/cosign` | Cosignature endpoint. Body is a `cosign.WireRequest`. See `github.com/clearcompass-ai/attesta`. |
| GET    | `/healthz`   | Liveness probe. Returns `ok` with 200 once the cosign handler is wired and listening.             |

## Graceful shutdown

`SIGINT` / `SIGTERM` triggers `http.Server.Shutdown` with a 5s
deadline. In-flight cosign requests complete; new connections are
rejected.

## Test

```sh
# Fast feedback (unit tests only):
go test -short ./...

# Full suite (includes the daemon e2e test that builds + execs the binary):
go test ./...
```

## Module boundary

This module imports only `github.com/clearcompass-ai/attesta`. It
never imports `github.com/clearcompass-ai/ledger` or any other
project repository — the witness daemon is intentionally isolated
from ledger application state.
